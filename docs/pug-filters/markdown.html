Pug filters

:markdown

The :markdown filter transform markdown to HTML and highlights code syntax.

Enable filter

The :markdown filter require the markdown-it module.

npm install -D markdown-it

Defaults, the filter is disabled. If this filter is used, it must be enabled in webpack.config.js.

Enable default markdown

Use the Pug loader options in the webpack config:
{
  test: /.pug$/,
  loader: '@webdiscus/pug-loader',
  options: {
    // enable embedded filters
    embedFilters: {
      // enable :markdown filter
      markdown: true,
    },
  },
},
If the option is boolean true or {} then is used default markdown-it options.
Note: Defaults the highlighting of code blocks is disabled.

Enable markdown with language prefix

The option langPrefix {string} is CSS language prefix for pre tag. Can be useful for external highlighters used in browser.
Add the options in the webpack config: 
{
  test: /.pug$/,
  loader: '@webdiscus/pug-loader',
  options: {
    // enable embedded filters
    embedFilters: {
      // enable :markdown filter
      markdown: {
        langPrefix: 'language-'
      },
    },
  },
},

Enable markdown with highlighting

Highlighting requires the npm module:
npm install -D prismjs
To enable highlights code blocks in markdown add the highlight {object} option to config:
{
  test: /.pug$/,
  loader: '@webdiscus/pug-loader',
  options: {
    // enable embedded filters
    embedFilters: {
      // enable :markdown filter
      markdown: {
        // enable highlighting in markdown
        highlight: {
          verbose: true,
          use: 'prismjs',
        },
      },
    },
  },
},

The highlight options:

  • verbose {boolean} Enable output process info in console. Use it in development mode only. Defaults is false.
  • use {string} The name of a highlighting npm module. The module must be installed. Currently, is supported the prismjs only.
Note: if the highlight option is used then the langPrefix is ignored. The language prefix for pre tag will be determined by the used highlighting module.

Usage example:
:markdown
  ```md
  ## Hello world

  You can write text [with links](http://example.com) inline or [link references][1].

  * one _thing_ has *em*phasis
  * two __things__ are **bold**

  [1]: http://example.com
  ```

Hello world

You can write text with links inline or link references.

  • two things are bold

PHP

namespace LocationWeb;

abstract class URI extends BaseURI
{
    abstract function test();
    public static $list = [];

    /**
     * Returns a URI
     * @return URI
     */
    static public function _factory($stats = array(), $uri = 'http')
    {
        $uri = explode(':', $uri, 0b10);
        $scheme = isset($uri[1]) ? $uri[1] : '';

        // Security check
        if (!ctype_alnum($scheme)) {
            throw new UriException('Illegal scheme');
        }

        return [
            'uri'   => $uri,
            'value' => null,
        ];
    }
}
